## Protocol

1. Read `CONTEXT.md`, `.roles/ROLE.md`, and this file.
2. Read the active wave and current pulse.
3. Confirm the evidence layer and allowed scope.
4. Implement the smallest complete slice.
5. Record sources, role findings, validation, failures, and transfer limits.
6. Close only when the wave's decision gate is answered.
7. Name the next learning target and falsifiable experiment.

## Growth cycle

RATIFY rotates through four modes:

1. **Breadth** — discover mechanisms and jurisdictions.
2. **Depth** — verify rules, outcomes, and failure modes.
3. **Integration** — translate lessons into bounded U.S. designs.
4. **Adversarial validation** — simulate and role-review failure cases.

No mode may run indefinitely. A breadth wave must produce a depth priority; a
design wave must produce a falsifiable experiment; an experiment must feed a
design decision or be retired.
