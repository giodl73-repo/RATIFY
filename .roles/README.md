# RATIFY Review Roles

Use these roles for every evidence packet, design decision, experiment, and
readiness claim. A review records findings in the active wave; merely having a
role file does not constitute review.

The system borrows RUNE's useful discipline:

- physical, inspectable role files;
- a parliament of competing technical lenses;
- explicit stakeholder voices;
- bounded waves and retained pulses;
- a future-user check before declaring a surface ready; and
- evidence-backed gates between research, design, simulation, and adoption.

RATIFY adds an innovation loop: every completed wave records what was learned,
what failed, which outside system should be studied next, and the smallest
falsifiable experiment that follows.
